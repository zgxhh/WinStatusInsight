# WinStatusInsight 产品说明

## 产品定位

WinStatusInsight 是一个只在本机运行的 Windows 状态分析面板。它用于在电脑变卡时快速判断瓶颈来自 CPU、内存、磁盘，还是某些后台自启、服务或常驻应用。

面板不会上传数据，不会修改启动项。点击「清理」时会尝试结束对应进程；系统、安全、更新类进程会自动置灰禁用，避免破坏系统。

## 核心功能

1. 点击「读取状态」
   - 调用本地 PowerShell 脚本采样约 2 秒。
   - 读取 CPU、内存、磁盘忙碌度、进程列表、启动项和运行中服务。
   - 自动保存一份 JSON 快照到 `data/snapshots`。

2. 流畅评分
   - 满分 100。
   - CPU、内存、磁盘越接近满载，扣分越多。
   - 单个进程 CPU >= 15% 或内存 >= 1GB 会额外扣分。
   - 评分文案分为：流畅、轻微压力、明显卡顿风险、严重卡顿。

3. 高占用进程
   - 「CPU 高占用」按采样期间 CPU 增量排序。
   - 「内存高占用」按工作集内存排序。
   - 表格展示 PID、进程名、CPU、内存、风险、标签、路径。

4. 后台进程识别
   - 命中 Windows 启动项：标记为「后台自启」。
   - 命中运行中服务：标记为「后台服务」。
   - 路径在 Windows 系统目录或系统白名单：标记为「系统进程」。
   - 路径在用户目录、Program Files、桌面常用目录：标记为「用户应用」。
   - 非系统进程且资源较高，或后台来源明显但没有前台窗口：标记为「值得关注」。

5. 建议清理
   - 浏览器：建议关闭不用的标签页或扩展。
   - 网盘/同步：建议暂停同步，尤其避免扫描开发目录。
   - 开发工具：建议关闭不用的项目窗口、插件或本地服务。
   - 创作软件：建议保存后关闭暂时不用的工程。
   - 顶部「建议清理」只列出推荐清理且允许清理的应用进程。
   - 完整进程表格会显示所有进程的控制按钮；系统、安全、更新相关项自动置灰不可点击。
   - 对可清理项点击「清理」会弹出确认框，确认后调用本机 API 结束该 PID。

6. 历史对比
   - 支持读取 `data/snapshots` 中的历史快照。
   - 在下拉框中勾选两次读取结果进行对比。
   - 对比评分、CPU、内存变化。
   - 汇总同名进程，只列出主要影响流畅的应用级 CPU、内存和进程数量变化。
   - 支持读取旧版文本日志目录：`C:\Users\HUAWEI\Documents\Codex\2026-05-10\win\lag-logs`。

## 运行方式

在项目目录执行：

```powershell
npm install
npm run dev
```

然后打开：

```text
http://127.0.0.1:5273
```

也可以双击项目里的 `start-win-status-insight.bat`。

## 注意事项

- 部分系统进程路径或命令行可能因为权限不足无法读取，这是正常现象。
- 评分是排查辅助，不等同于硬件健康检测。
- 「值得关注」不是病毒判断，只表示该进程在当前状态下值得人工确认。
- 如果磁盘忙碌度长期接近 100%，通常比 CPU 更容易造成整机卡顿。
